# CI/CD Setup Guide: GitHub Actions → GCP Cloud Run

This guide explains how to activate the automated deployment pipeline for the Faculty Appraisal Backend.

Project:  facultyappraisal-495011
Region:   asia-south1
Service:  fastapi-backend
Registry: asia-south1-docker.pkg.dev/facultyappraisal-495011/cloud-run-source-deploy/fastapi-backend

---

## 1. GitHub Secrets Configuration

The pipeline is code-complete in .github/workflows/. It needs these secrets in GitHub to run.

Steps:
1. Open the repository on GitHub.
2. Go to Settings > Secrets and variables > Actions.
3. Click "New repository secret" for each entry below.

### Required secrets

| Secret Name       | Value / Description                                                      |
|-------------------|--------------------------------------------------------------------------|
| GCP_PROJECT_ID    | facultyappraisal-495011                                                  |
| GCP_REGION        | asia-south1                                                              |
| GCP_SA_KEY        | Paste the ENTIRE JSON content of the GCP Service Account Key file        |

### Cloud Run environment variables (set via --set-env-vars or Secret Manager)

These must be available to the running container. Add them either as GitHub secrets
passed to `gcloud run deploy --set-env-vars`, or store them in GCP Secret Manager
and reference them in the Cloud Run service configuration.

| Variable             | Description                                                        |
|----------------------|--------------------------------------------------------------------|
| DATABASE_URL         | postgresql+asyncpg://user:pass@host:6543/postgres (Supabase pooled)|
| JWT_SECRET_KEY       | Long random string for signing tokens                              |
| USE_LOCAL_AUTH       | true (use built-in auth) or false (Supabase Auth)                  |
| APP_URL              | Public URL of this Cloud Run service (no trailing slash)           |
| FRONTEND_URL         | Netlify frontend URL (no trailing slash)                           |
| SMTP_HOST            | smtp.gmail.com                                                     |
| SMTP_PORT            | 587                                                                |
| SMTP_USER            | Sending email address                                              |
| SMTP_PASSWORD        | Gmail App Password (not regular password)                          |
| MAIL_FROM            | Sending email address                                              |
| GCP_STORAGE_BUCKET   | GCS bucket name for file uploads                                   |
| GCP_PROJECT_ID       | facultyappraisal-495011                                            |
| SUPABASE_URL         | Only needed if USE_LOCAL_AUTH=false                                |
| SUPABASE_ANON_KEY    | Only needed if USE_LOCAL_AUTH=false                                |

## 2. GCP Service Account Permissions

The service account used in GCP_SA_KEY needs these IAM roles:

1. Artifact Registry Writer     — to push Docker images
2. Cloud Run Developer          — to deploy/update the Cloud Run service
3. Service Account User         — to act as the service account identity during deploy

---

## 3. One-Time GCP Setup

Before the first pipeline run:

1. Create Artifact Registry repository:
   - Name:   cloud-run-source-deploy
   - Format: Docker
   - Region: asia-south1

2. Enable these APIs in the GCP project:
   - Cloud Run API
   - Artifact Registry API
   - Cloud Build API

---

## 4. How the Pipeline Works

Every git push to the main branch:
1. GitHub Actions triggers the workflow.
2. Builds a Docker image from the Dockerfile in the repo root.
3. Pushes the image to Artifact Registry tagged with the commit SHA.
4. Runs: gcloud run deploy fastapi-backend --image <new-image> --region asia-south1
5. Cloud Run swaps to the new revision with zero downtime.

---

## 5. Manual Deploy (Without GitHub Actions)

If you need to deploy manually from your local machine:

  cd Faculty_appraisal
  git pull

  export PROJECT_ID=facultyappraisal-495011

  gcloud builds submit \
    --tag asia-south1-docker.pkg.dev/$PROJECT_ID/cloud-run-source-deploy/fastapi-backend .

  gcloud run deploy fastapi-backend \
    --image asia-south1-docker.pkg.dev/$PROJECT_ID/cloud-run-source-deploy/fastapi-backend \
    --region asia-south1 \
    --allow-unauthenticated

Full command reference is also in gcp_deploy_codes.txt at the project root.

---

## 6. Checking Logs After Deploy

  gcloud run services logs read fastapi-backend --region asia-south1 --limit 50

Or open Cloud Console > Cloud Run > fastapi-backend > Logs.

Look for "shred_form:" INFO lines after a faculty submission to verify rows are being written.
