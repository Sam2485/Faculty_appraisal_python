⚠️ Critical Warnings for Restoring
Duplicate Data: If your database still has data in it, running an import might fail or create "duplicate" errors (because it tries to create tables that already exist). Before a full restore, it is often best to drop the existing database and create a fresh, empty one:

# WARNING: This deletes current data to make room for the backup
gcloud sql databases delete faculty_appraisal --instance=faculty-appraisal-db
gcloud sql databases create faculty_appraisal --instance=faculty-appraisal-db
